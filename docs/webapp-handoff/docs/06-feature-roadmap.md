# Feature Roadmap

## Phase A1 — Android handoff/context package

- create this reference package
- collect the current webapp route and data model context
- define the mobile architecture and API direction

## Phase A2 — Kotlin app foundation

- set up the Android project structure
- wire Compose navigation and screen scaffolding
- add base networking, theming, and local storage layers

## Phase A3 — Auth/session

- sign in and session handling
- bearer-token API access
- local session metadata in DataStore

## Phase A4 — Program search/list/detail

- program discovery
- program detail screens
- search filters and sorting

## Phase A5 — Saved items + offline cache

- saved item list
- save and unsave actions
- Room cache for offline reading and recent content

## Phase A6 — Universities/scholarships/guides

- university detail screens
- scholarship detail screens
- guide/article reading screens

## Phase A7 — Fit Finder native flow

- native Fit Finder questionnaire
- server-scored result generation
- saved Fit Finder result detail screens

## Phase A8 — AI chat native UI

- chat conversation screen
- contextual help and answers
- server-backed safety and quota handling

## Phase A9 — Deadline reminders/push notifications

- background sync jobs
- deadline or intake reminders
- push notification delivery later

## Phase A10 — Contributor profile support later

- contributor-facing profile state if needed
- avatar or public attribution support if the mobile product requires it later
- no need to build this before the core student app
